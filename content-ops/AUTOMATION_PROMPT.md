# ChinaFreeWeight daily multilingual publishing protocol

Work only in the automation worktree. Use the current Asia/Shanghai date. Run `npm run content:acquire` first and always run `npm run content:release` in a final cleanup step. Run `npm run content:init`.

Before writing, scan the live site, sitemap, repository content and the private registry topic ledger. For every public locale, research real product keywords, long-tail procurement queries and buyer questions on current search platforms. Inspect the organic top ten results and relevant competitors, record what they cover, why their result is useful, their evidence and decision-support patterns, and what ChinaFreeWeight lacks. On Sunday, refresh the paid research cache only if credentials are available and the combined weekly spend remains at or below US$15; otherwise use public research and record the limitation internally.

Build at least 15 distinct candidates per locale, then select up to five with the strongest B2B purchase relevance and genuine site gap. Stay within dumbbells, weight plates, Olympic bars, cable handles, free-weight accessories and OEM free weights. Write for importers, distributors, wholesalers, fitness brands and procurement managers. Every article must be original, locally natural, technically deep, decisive, source-traceable and useful for specification, supplier comparison, landed-risk control or purchasing. Do not translate a template. Do not reuse an outline, opening, examples or FAQ set across articles or sites.

Visitor-facing copy must never mention optimization work, ranking tactics, machine generation, uncertainty, placeholders or facts awaiting confirmation. Never invent specifications, certifications, customers, tests, capacity, lead times, pricing or factory evidence. Author is exactly `ChinaFreeWeight Technical Editorial Team`.

Each draft in `content-ops/runs/YYYY-MM-DD/drafts/index.json` must follow the shared QA document contract: direct answer, at least six substantive sections and twelve paragraphs, a decision table/framework/checklist/matrix, exactly five buyer FAQs, four to eight same-language internal links, at least three traceable sources including one primary source, and at least one relevant image with `src`, `alt`, `provenance` and `rights`. Add target keyword, intent, buyer role, related questions and proposed public path.

Run `npm run content:qa`. Publish only IDs marked `passed`; never bypass the gate to reach five. Convert each passing document into the existing native site content model, preserving canonical, hreflang, Article/FAQ/Breadcrumb schema, index pages, sitemaps and internal links. Verify typecheck, tests and production build, then commit and push the automation branch and merge only if all checks pass. After successful deployment verification, set `publishedPath` and `publishedAt`, run `npm run content:record`, commit/push the private registry ledger, and report per-locale published/withheld counts plus exact errors.
